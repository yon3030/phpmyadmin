Failure of Success Criterion 2.1.4 due to implementing character key shortcuts that cannot be turned off or remapped

Metadata

ID: W##

Technology: General

Type: Failure

Applicability

This failure applies to web content that can be interacted with using a physical keyboard as input mechanism.

The Failure relates to:

  • Success Criterion 2.1.4 Character Key Shortcuts

Description

Character key shortcuts using a single key such as a letter key (including upper- and lower-case letters), punctuation, number, or symbol characters can cause problems for people who use voice input on a computer or mobile device. When character key shortcuts are implemented, voice commands can inadvertently trigger functions, and the context of use is lost.

Content must either not implement single character key shortcuts or offer settings to either turn off these shortcuts or to map them to keyboard shortcuts that employ an additional modifier key, like ALT or STR.

Note: The use of a single key keyboard shortcut is not a Failure if the shortcut is only active when a particuar interface element has focus. For example, when a select element or a custom listbox has focus, the input of single character keys to navigate the list is a useful feature.

Examples

Example 1

An application uses the single key shortcut S to bring up a search popup (looking for a realistic example here). There is no setting available to turn off or modify the shortcut.

Tests

Procedure

  1. Reload page. If loading the page sets focus to an input, click on an empty part of the page to ensure that no inputs are in focus.
  2. Press all printing characters, i.e. all number, letter, sign and punctuation keys. Do not press non-printing modifier and control keys such as STR, ALT, ESC, Arrow keys and (where present) the function keys F1-F12. Also exempt are space, ENTER, Return, Tab, and the Delete key.
  3. Hold the SHIFT key and press the same keys again.
  4. If the pressing of single character keys triggers any functions, check whether the website or app provides settings to disable keyboard shortcuts using single character keys or map them to keyboard shortcuts that use a modifier key.

Notes

The success criterion does not apply when single key shortcuts are only active when interface elements have the focus, for example, a select element. Here, pressing a letter key is used for fast navigation within the select options.

Viewing page scripts and searching for typical keyboard event handlers like document.addEventListener('keydown' ...) or the presence of the .keycode attribute may establish the presence of scripts that intercept keyboard shortcuts without modification keys like ALT or STR being held down at the same time. As there are several ways of implementing character key events, this method is not considered reliable.

Some browsers employ single key shortcuts with SHIFT. For example, Firefox opens a page search when pressing SHIFT + / and a search in page links when pressing SHIFT + '. In these cases, it will be necessary to press ESC or click an empty part of the page to remove the focus from the browser input.

Expected Results

  • No changes are triggered to the page content or context by pressing single character keys.
  • There are keyboard shortcuts via single character keys, but these can be switched off via settings of the website or app or mapped to key combinations with modifier keys.
