Using HTML 5.2 autocomplete attributes

Metadata

html

sufficient

When to Use

This technique can be used on all form fields that map to the HTML 5.2 autofill tokens.

Description

The technique works by adding the appropriate autocomplete token to each form field on the form. This will help people with cognitive disabilities who may not immediately know the purpose of the field because the label used by the author is not familar to them, those who have trouble typing such those with dexterity disabilities, and generally helps anyone who needs more time to fill out a form.

The autocomplete attribute improves the browser's natural ability to remember form fields by explicity mapping it to a purpose for the field. It reduces typing by filling in details, it reduces reliance on memory because it uses stored values. It also gives third party plugins and software the ability to manipulate form fields to make them more accessible to people with cognitive disabilities. For instance, a plugin could sniff for an autocomplete token with the text string "tel" and insert a telephone icon. It will further enable third party software to swap out the labels for a text label that the user finds more familiar. For instance, it could change "Given Name" to "First Name".

It's important to note that the success criteria only places requirements on input fields collecting information about the user.

The objective of this technique is to make form fields easier to fill out and understand.
