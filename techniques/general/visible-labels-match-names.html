Description

The objective of this technique is to help people who use speech input to operate web content reliably. The secondary objective is to prevent speech users from accidentally activating controls.

When speech users interact with a web page,, they usually speak a command followed by the name of a label (like a button or a link), for example, "click search". The speech recognition software can then process the text it has understood and apply the command to the control (such as activating an input of type=submit with the value "search").

The problem addressed by this technique is that the name of a control that speech recognition sodtware used for this matching (the so-called "accessible name") is not necessarily identical to the visible label. For example, a button follwing a search input field may contain the text "go" but use an invisible aria-label attribute with the value "search". Since aria-label takes precedence over the text included in the button, the accessible name of this button is "search", not "go". (Compare the Accessible Name and Description Computation algorithm).
