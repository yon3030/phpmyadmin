The objective of this technique is to help people who use speech input to operate web content to reliably activate controls, and at the same time prevent users from unwillingly activating controls.

When used to interact with a web page, speech users usually say a command followed by the name of a label (like a button or a link) such as "click search". the speech recognition software can the match the text it has understood to the name of a control to apply the command (such as activating an input of type=submit with the value "search"). The problem this technuique addresses is that the name of a control used for this matching (th eso-called "accessible name") is not necessarily the visible label. For example, the button follwing a search input field may contain the text "go" but use an invisible aria-label attribute with the value "search". Since aria-label takes precedence over the text included in the button, the accessible name of this button is search, not go. (Compare the Accessible Name and Description Computation algorithm).
