Failure due to accessible name not containing the visible label text

Metadata

general

failure

When to Use

This Failure applies to all web technologies that include interactive controls (such as links or form inputs)

Description

The objective of this Failure is to capture situations where speech input users cannot operate web content reliably.

When speech input users interact with a web page, they usually speak a command followed by the reference to some visible label (like the text in a button or a link, or the text labelling some input). If the visible label does not match the accessible name (or the beginning of the accessible name) of the control, speech users may be unable to directly activate that control.

When techniques such as the use of accessibly hidden text, aria-label, or aria-labelledby attributes are used to add to the visible label text in order to provide more context, and this addition is placed before the visible label text, speech users may equally be unable to directly activate that control. (Refer to the Accessible Name and Description Computation algorithm to work out the order of precendence in computing the accessible name of a control.)

Examples

Failure example 1: Mismatch of visible button text and accessible name supplied via aria-label

The text in a search button reads "Go" but the accessible name provided in an aria-label attribute is "Find in this site"


        <button id="sitesearch" aria-label="Find in this site">Go</button>

Failure example 2: Invisible link text part of the accessible name

A download link reads "Download specification" but there is invisible link text so that the accessible name of that link is "Download gizmo specification". While the visble label text is contained in the accessible name, there is no string match which may prevent the link from being activated by speech input.


        <a href="#">Download <span class="accessibly-hidden">gizmo</span> specification</a>

Failure example 3: Button with a hidden label carrying different text than visible text

A button element enclosing the text "search" has a programmatically linked and accessibly hidden label enclosing the text "Find in this site". Because label takes precedence over the enclosed text content of the button, the accessible name of the button in most browser / screen reader combinations will be "Find in this site". Speech users speaking a command such as "Click search" will be unable to activate the control.


        <div id="hidden-label">Find in this site</div>
        <button id="sitesearch" aria-labelledby="hidden-label">Search</button>

Tests

Procedure

For all controls with a visible label (e.g. link text, button text, programmatically linked label, value of input), check that

  1. The visible label differs the accessible name of the control, or
  2. The visible label string is included in the accessible name of the control, but in parts which are in a different order or interspersed with other words

Expected Results

  • If step #1 or step #2 are true, the content fails the Success Criterion.
