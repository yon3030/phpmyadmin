Using CSS media queries showing a message asking to reorient device

Metadata

css

sufficient

When to Use

This technique is applicable to Cascading Style Sheet / HTML technologies.

Description

The objective of this Failure is to describe the situation where users are unable to rotate their device because it is mounted in a fixed orientation (e.g., on the arm of a power wheelchair).

If web content is designed for portrait only and when landscape is detected, a message appears asking to reorient the device, these users will be unable to rotate and operate the content.

There are various methods for devices to determine if the content is in landscape or portrait orientation. One of these methods is by looking at the width-to-height aspect ratio of the viewport. In other words: if the width is smaller or larger than the height. The CSS orientation media feature is portrait when the value of the height media feature is greater than or equal to the value of the width media feature. Otherwise orientation is landscape.

Examples

Block a orientation, "door slam", by using HTML and CSS

The following example uses HTML and CSS to show a message asking to reorient the device if in landscape.

The message will disappear if the orientation is changed to portrait. When the portrait property matches the viewport space defined via media queries, the message will disappear or vice versa. This particular example uses the CSS landscape, portrait media query properties.



<style>

/* Orientation "door slam" Styling */

@media all and (orientation: landscape) {
  .rotate { display: block; }
  .content { display: none; }
}

@media all and (orientation: portrait) {
  .rotate { display: none; }
  .content { display: block; }
}

.rotate {
  text-align: center;
  font-size: 2rem;
  margin-top: 4rem;
}

/* Orientation "door slam" HTML */

<div class="rotate">
  Please rotate your device!
</div>

<div class="content">
  ...
</div>
