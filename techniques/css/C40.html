Creating a two color focus indicator to ensure sufficient contrast with all components

Metadata

HTML and CSS

When to Use

Components on a page are often many colors and shades. Historically focus indicators have been one color, so they are highly visible when some components have focus and not well seen on other components. For instance, if a focus indicator is dark blue and a button is yellow then it will be well seen, but if the button is also blue it will not be well seen. This technique overcomes that problem with one class that can be used across a site.

Description

Although it is possible to create individual CSS classes to address the different buttons across a site, this can be time consuming and easy to miss some types of interactive content. However, if the focus indicator is two colors, a light color and a dark color, then regardless of the color of component it is on, it will always have sufficient contrast. Currently, this can be done by combining the text shadowproperty with the outline property on the focus indicator.

The objective of this technique is to create a two color focus indicator that is always visible and does not require multiple classes to ensure constant sufficient contrast of the focus indicator regardless of the color of the component it is focused on.

Tests

Procedure

  1. Step 1: Load a page with light and dark interactive elements and/or light and dark backgrounds
  2. Step 2: Press the tab key to move the focus ring between the elements

Expected Results

  • The focus indicator is always be visible and have a sufficient contrast ratio of 3:1 with adjacent components as the user tabs through the page. The light color is visible on a dark components or backgrounds and the dark color is visible a light colored components or backgrounds
