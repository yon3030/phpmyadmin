Using media queries to un-fix sticky headers / footers

Metadata

css

sufficient

When to Use

This technique is applicable to Cascading Style Sheet / HTML technologies.

Description

The objective of this technique is to be able to present content without sticky headers and footers when the device is held in landscape mode. This is done by using orientation techniques that adapt to the portrait or landscape mode of the device.

Sticky regions always stay visible in the viewport while the other content will disappear underneath when scrolling. This often is no problem in portrait mode but may block a big portion of the screen in landscape mode because of the height of the regions and leave only a small part of the screen for users to profit from. This is an effective way to create designs that present well in portrait mode while when users prefer landscape mode because of different reading and zoom preferences the sticky regions will be disabled.

The basic principle of un-fix sticky headers / footers is to:

  1. Define the sticky regions using orientation properties or properties for specific viewport sizes and media queries, so they get fixed or un-fixed depending on the portrait or landscape position of the device;

Examples

Example 1: Medium complex un-fix sticky headers / footers in HTML and CSS

The following medium complex example uses HTML and CSS to un-fix sticky headers / footers. The sticky regions get un-fixed as the orientation is changed. When the orientation property matches the device orientation defined via media queries, regions which are sticky in portrait mode will un-fix in landscape mode or vice versa. This particular example uses the CSS "orientation: portrait" property while it is also possible to use specific height properties.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Using media queries to un-fix sticky headers / footers</title> <style> /* Sticky Styling */ @media screen and (orientation: portrait) { header { position: sticky; top: 0; } footer { position: sticky; bottom: 0; } } </style> </head> <body class="grid"> <header role="banner" class="grid-item"> ... </header> <main role="main" class="grid-item"> ... </main> <aside role="complementary" class="grid-item"> ... </aside> <footer role="contentinfo" class="grid-item"> ... </footer> </body> </html>

Tests

Procedure

  1. Display content on a mobile device / user agent in portrait mode.
  2. Change the orientation to landscape.
  3. Check whether the sticky header and footer will be un-fixed.

Expected Results

  • Check #3 is true.
