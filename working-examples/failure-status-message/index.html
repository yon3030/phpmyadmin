Including a search results message without a status role

This demonstrates how messages about search results are returned without being surfaced using the aria status role. After a user presses a Search button, the page content pretends to do a search for words that rhyme with "orange". None are returned, which results in a change to content, the message "0 results returned". This text is not given an appropriate role for a status message, and so a screen reader will not announce "Zero results returned" without the new content being given focus.
