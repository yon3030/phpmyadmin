Input Purposes for User Interface Components

This section contains a listing of common user interface component input purposes. The terms below are not keywords that must be used, but instead represent purposes that must be captured in the taxonomy adopted by a webpage. Where applicable, authors mark up controls with the chosen taxonomy to indicate the semantic purpose. This provides the potential for user agents and assistive technologies to apply personalized presentations that can enable more people to understand and use the content.

The list of input type purposes is based on the control purposes defined in the HTML 5.2 Autofill field section, but it is important to understand that a different technology may have some or all of the same concepts defined in its specification and only the concepts that are mapped to the meanings below are required.

Common Input Control Purposes

The following input control purposes are intended to relate to the user of the content and pertain only to information related to that individual.

  • Name - Inputs used to handle information about a userâ€™s name(s) (e.g. first name, family name, suffix, etc.)
  • Professional Title - Job title (e.g., "Software Engineer", "Senior Vice President", "Deputy Managing Director")
  • Organization - Company name corresponding to the user
  • Address - Inputs used to handle address information (e.g. street address, city, region, postal code, etc.)
  • Country Code - A country's abbreviation code
  • Country Name - Full name of a country
  • Credit Card - Inputs used to handle information about credit card payments (e.g. credit card holder name, card number, Security code, card type, etc.)
  • Transaction Currency - The monetary system being used for the transaction
  • Transaction Amount - The amount that the user is committing to for a transaction (e.g., when entering a bid or sale price)
  • Language - Preferred language
  • Birthdate - Inputs used to handle information about a user's birthdate
  • URL - Home page or other Web page corresponding to the user
  • Photo - Photograph, icon, or other image corresponding to the user
  • Telephone Number - Inputs used to handle information about telephone numbers
  • Email Address - Email address
